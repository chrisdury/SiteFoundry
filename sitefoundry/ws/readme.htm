XStandard Web Services For ASP.NET Readme File

Introduction To XStandard Web Services

Web Services are applications that run on server computers and communicate with other computers using a dialect of XML called SOAP (Simple Object Access Protocol). Typically, business users do not interact directly with Web Services. Instead, they interact with user-friendly programs (such as XStandard) that in turn use Web Services. XStandard uses Web Services to check spelling, to upload files from the local computer to the server and to reference files located on remote servers.

XStandard Pro works with 5 Web Services - SpellChecker, ImageLibrary, AttachmentLibrary, Directory and Subdocument. SpellChecker is used for spell checking. ImageLibrary is used to upload images from a local computer to the server and to build and browse image libraries. AttachmentLibrary is used to upload files from a local computer to the server and to build and browse file libraries. Directory is used to communicate with 3rd party systems (like your CMS) and insert code snippets into the editor. Subdocument is used to replace custom elements with subdocuments.

Requirements

  • Microsoft .NET Framework 1.1 Version 1.1.4322 SP1 or greater. Some examples require MDAC 2.6+.
  • Windows NT+

Installation

  1. Unzip x-web-services-aspx.zip
  2. Move spellchecker.aspx, spellchecker.config, imagelibrary.aspx, attachmentlibrary.aspx, linklibrary.aspx, imagelibrary.config, attachmentlibrary.config, directory.aspx, directory.config, directory-example-product-1.txt, directory-example-product-2.txt, directory-example-product-3.txt, directory-example-product-4.txt, directory-xhtml-entities.xml, directory-xhtml-latin1.xml, directory-xhtml-special.xml, directory-xhtml-symbol.xml, directory_example_staff.csv, subdocument.aspx, subdocument-example-A.txt, subdocument-example-B.txt, subdocument-example-C.txt, subdocument-example-D.txt and subdocument-example-E.txt to a virtual root.
  3. Copy the bin, data and dict folders to a sub-folder that contains the ASPX files.
  4. In IIS, make the folder containing the ASPX files into an Application. If you are not able to create an Application because your are in a shared hosting environment, then put the ASPX files into the root folder of your Web site.

Testing

Open a Web browser and for each Web Service navigate to the URL where the Web Services is located. For example:
http://localhost/imagelibrary.aspx
http://localhost/attachmentlibrary.aspx
http://localhost/directory.aspx
http://localhost/spellchecker.aspx
http://localhost/subdocument.aspx
If you don't see Status: Ready in the browser, then the Web Service is not correctly installed. The most common cause of this is incorrectly set file permissions.

Launch test file test.htm. Specify the path to the license file and to Web Services. Click on the "Update" button. You should now be able to reference files from a library, drag an image into the editor, or spell check the editor's content.
